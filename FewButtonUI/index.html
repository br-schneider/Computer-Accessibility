About this app

Tested 3-key vs 5-key Interaction

  • 3 - key version
    A key = left/previous
    D key = right/next
    S key = click selected button


    5 - key version (currently implemented)
    W key = up
    A key = left
    S key = down
    D key = right
    Q key = click selected button (didn't know if this would be the best select key)


  • Did the set operations below 5 times each and calculated the average time of each of them would take.
    It was interesting because with more keys I felt like I could move faster but then made a lot of errors.
    With the 3 keys I moved slower and more precise because it didn't feel as familiar I believe. Another thing
    I noticed was by transitioning from 3 key to 5 key I made some errors in the 1st test of the 5 key because
    I got used to the 3 key. I also had a thought that maybe if I made the 5 key work so that I did movement with
    the left hand and selection with the right hand maybe it would have felt more natural. I think it was good however
    to get the consistency of using one hand for both tests. Overall the results were much different than I
    thought they would be.

    3 key test results and operations
    5*5+3-2 --- (20 seconds, 1 error)
    1+2+3+4+5 --- (16 seconds, 0.5 errors)
    2*8 --- (8 seconds, 0 errors)
    5+3-2 --- (13 seconds, 0.8 errors)
    10/20/5 --- (19 seconds, 0.6 errors)


    5 key test results and operations
    5*5+3-2 --- (14 seconds, 2.2 errors)
    1+2+3+4+5 --- (15 seconds, 2 errors)
    2*8 --- (4 seconds, 0.2 errors)
    5+3-2 --- (9 seconds, 1 error)
    10/20/5 --- (13 seconds, 1.6 errors)
